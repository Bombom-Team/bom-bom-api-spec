# Issue Tracker

This repo uses **Jira** for issue tracking and project management.

## Ticket format

When creating or proposing issues, use the following structure:

- **Summary** — one-line, action-oriented title
- **Background** — context behind the work
- **Problem** — what's broken or missing
- **Proposed Solution** — concrete implementation direction
- **Acceptance Criteria** — bullet list of verifiable conditions
- **Priority** — Highest / High / Medium / Low / Lowest
- **Labels** — relevant Jira labels (e.g. `backend`, `infrastructure`, `api`)

Keep tickets concise, implementation-oriented, and suitable for backend/infrastructure workflows.

## Notes

- There is no CLI integration configured. Proposed tickets should be written in Jira-friendly format and pasted into Jira manually, or via your team's preferred integration.
